## üë• Manage Users

### User Basics

Each user has:

- **Username**
    
- **Password**
    
- **One Profile** (defines defaults like login hours)
    
- Optional: **Permission Sets** / **Permission Set Groups** for extra access
    

### View & Create Users

1. Go to **Setup** ‚Üí search **"Users"**
    
2. Click **New User** or **Add Multiple Users**
    
3. Enter:
    
    - Name, Alias, Email, Username (must be unique)
        
    - User License ‚Üí determines available Profiles
        
    - Select a Profile
        
    - Choose to auto-generate & send password
        
4. Click **Save**
    

### Deactivate or Freeze Users

- You **can‚Äôt delete** users, but you can deactivate or freeze them:
    
    - **Deactivate** = user loses all access
        
    - **Freeze** = temporarily prevent login (e.g. before deactivating)
        

> üîç Setup ‚Üí Users ‚Üí Edit ‚Üí Uncheck **Active**  
> üîç Or click **Freeze** on user detail page

## üîë Set Password Policies

### Configure from:

**Setup** ‚Üí Search **"Password Policies"**

You can set:

- **Min password length**
    
- Required **character types**
    
- **Expiration timeframe**
    
- **Login attempts before lockout**
    
- What happens when passwords are forgotten
    

You can also:

- **Reset passwords** for specific users
    
- **Unlock** users after failed attempts
    
- **Expire all passwords** (except those with ‚ÄúPassword Never Expires‚Äù)
    

üìå **Note:** Profile-level password policies can override org-wide settings.

## üåê IP Address Restrictions

### Org-Wide Trusted IP Ranges

Allows users to skip verification code when logging in from trusted locations.

**Setup** ‚Üí Search **"Network Access"** ‚Üí Click **New**  
‚Üí Add **Start IP** and **End IP** ‚Üí Save

---

### Profile-Based Login IP Ranges

Completely restricts users with a certain profile to **only log in from certain IPs**.

**Setup** ‚Üí Search **"Profiles"** ‚Üí Select Profile  
‚Üí Under **Login IP Ranges**, click **New**  
‚Üí Enter IP range ‚Üí Save

> No verification code option here‚Äîusers are either in or out.

## üïí Login Hour Restrictions

### Limit logins to specific times per profile:

**Setup** ‚Üí Search **"Profiles"** ‚Üí Select Profile  
‚Üí Under **Login Hours**, click **Edit**  
‚Üí Set login hours for each day

- To block access on a day, set start = end time
    
- To allow 24/7 access, click **Clear all times**
    

> ‚ö†Ô∏è If users are logged in when time ends, they can still **view**, but not take **action**.
